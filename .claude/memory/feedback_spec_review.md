---
name: Skip spec review gate
description: User doesn't want to manually review long spec documents — trust the process and move forward
type: feedback
---

When asking the user to review a spec document, don't expect them to read the full thing. They trust the brainstorming process and want to keep moving. Summarize key decisions in a few bullets if needed, but don't gate on "please review this doc."

**Why:** User prefers momentum over thoroughness on review steps. Reading long specs feels like friction.

**How to apply:** After spec is written and passes automated review, move straight to implementation planning unless the user explicitly asks to review.
